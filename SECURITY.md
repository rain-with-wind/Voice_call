# 安全策略

## 报告漏洞

如果你发现安全漏洞，请通过 GitHub Issues 私下报告，**不要**公开披露。

请包含以下信息：

- 漏洞描述
- 复现步骤
- 受影响版本
- 可能的修复建议

我们会尽快确认并在修复后发布更新。

## 安全模型

本项目采用轻量级安全设计：

- **认证**：基于 SHA-256 哈希的共享密码认证
- **加密**：基于 PBKDF2 密钥派生 + Fernet 对称加密
- **密钥交换**：服务端生成随机 salt，双方各自派生相同密钥

### 局限

- 不提供前向安全性（Forward Secrecy）
- 不包含身份证书或公钥基础设施（PKI）
- 密码通过命令行参数传递，可能被系统进程列表暴露
- 适用于个人项目、教学和内部实验，**不适用于对安全性有严格要求的场景**

## 最佳实践

- 使用足够复杂的密码
- 在可信网络环境中使用
- 通话结束后密码不会保留在系统中（驻留在内存中的密钥会在进程退出时清除）

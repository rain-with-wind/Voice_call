# 行为准则

本项目希望保持一个友好、尊重和专注的协作环境。无论是提交 Issue、参与讨论，还是提交 Pull Request，都请遵守以下基本规则。

## 基本原则

- 尊重不同经验水平的参与者
- 讨论问题时围绕事实和代码本身
- 提建议时尽量说明原因和可行方案
- 接受他人指出问题，并愿意改进
- 不发布攻击、歧视、骚扰或无关内容

## 鼓励的行为

- 清楚描述问题和复现步骤
- 认真阅读已有文档和讨论
- 对代码改动给出必要说明
- 对他人的贡献表达感谢
- 在发现错误时帮助定位原因

## 不接受的行为

- 人身攻击或嘲讽
- 歧视性、侮辱性语言
- 故意刷屏或发布无关内容
- 未经允许公开他人隐私信息
- 明知有风险仍引导他人执行危险操作

## 处理方式

维护者可以根据情况提醒、隐藏不合适内容、关闭讨论或拒绝合并相关提交。严重影响项目协作氛围的行为可能会被限制参与。
